# Claude Code 日本の政策・国政調査向け 最終版スターター

これは、**日本の国政・行政・政策・政治問題の調査**を Claude Code で進めるための、日本語ベースの最終版スターターです。

このスターターが前提にしているのは、**国政は永田町・霞が関の中だけで完結しない**という考え方です。したがって、官邸・国会・省庁・審議会を優先しつつ、必要に応じて次も拾います。
- 事故、事件、災害、消費者被害、技術変化、市場変化
- 企業・プラットフォーム・業界団体の対応
- 自治体の先行措置、条例、要望
- 裁判、仮処分、第三者委員会報告
- 海外当局、国際機関、外国議会の動き
- 被害者団体、NPO、学会、研究者、調査報道

この最終版は、次の 10 テーマを念頭に置いて、多様なテーマで使えるように調整しています。
1. 少子化対策・子育て支援
2. 物価高対策・家計支援
3. 安全保障・防衛・防衛産業
4. 外国人労働者・移民政策
5. 生成AI・プラットフォーム規制・デジタル政策
6. 医療・介護・社会保障改革
7. エネルギー政策・原発・再エネ・GX
8. 教育政策・無償化・大学改革・いじめ対策
9. 農業政策・食料安全保障・コメ政策
10. 災害対応・防災・復興政策

## この一式に含まれるもの
- `CLAUDE.md` — プロジェクト全体の恒常ルール
- `.claude/skills/` — 再利用できる調査ワークフロー
- `.claude/agents/` — 専門化されたサブエージェント定義
- `.claude/settings.json` — プロジェクト単位の設定と hooks
- `.claude/hooks/` — 軽いガードレールとログ
- `schemas/` — JSON Schema
- `docs/` — 日本の国政調査ガイド、10テーマ整理、運用ガイド、実装メモ
- `data/processed/` — 構造化済みの正本データ置き場
- `outputs/` — 可視化や QA レポート出力先

## 重要な注意
- **スキル名やサブエージェント名は英小文字ハイフン**のままです。これは Claude の Skill / subagent の仕様に合わせたものです。
- ただし、**中身の説明、運用ルール、出力の方針はすべて日本語**です。
- 国政調査は名誉毀損・誤情報・法的リスクが高いので、公開前に人間が最終レビューしてください。

## まずやること
1. このディレクトリを Claude Code で開く。
2. `CLAUDE.md` を読む。
4. `docs/日本の国政調査ガイド.md` と `docs/運用ガイド.md` を読む。
5. `/intake-source-batch <テーマ>` から始める。
6. 収集後に `/collect-events <テーマ>`、`/link-events <テーマ>`、`/qa-event-db <テーマ>`、`/build-timeline-view <テーマ>` の順で進める。

## データ配置
- `data/raw/` — 生の PDF、メモ、コピーしたテキスト
- `data/processed/events.ndjson` — イベント正本
- `data/processed/entities.ndjson` — 人物・組織などのエンティティ
- `data/processed/source_links.ndjson` — イベントごとの資料リンクと要約
- `data/processed/event_links.ndjson` — イベント間関係
- `outputs/` — HTML, JSON, CSV, QA レポート
