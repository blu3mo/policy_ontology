# 運用ガイド

## 基本の流れ

### 1. まず資料を広く集める
例:
`/intake-source-batch 生成AI・プラットフォーム規制と日本の政策形成`

### 2. イベント正本を作る
例:
`/collect-events 生成AI・プラットフォーム規制と日本の政策形成`

### 3. イベント間リンクを張る
例:
`/link-events 生成AI・プラットフォーム規制と日本の政策形成`

### 4. QA をかける
例:
`/qa-event-db 生成AI・プラットフォーム規制と日本の政策形成`

### 5. 可視化を作る
例:
`/build-timeline-view 生成AI・プラットフォーム規制と日本の政策形成`

## 最初に Claude へ渡すときの中核プロンプト

以下のテーマについて、日本の政策・国政調査として進めてください。官邸、国会、各省庁、審議会、規制当局、裁判所、政党、議員を優先しつつ、それだけに限定せず、自治体、企業、業界団体、当事者団体、研究者、海外当局、国際機関、調査報道も広く見てください。可能な限り一次資料を優先し、イベントごとに資料と日本語要約を付けてください。制度過程の外側で起きた出来事も、政策形成に影響するなら独立イベントとして記録してください。最後にイベント間リンクと時系列可視化まで進めてください。

## 10 テーマを念頭に置いた追加指示
- 「少子化、物価高、防衛、移民、生成AI、医療、エネルギー、教育、農業、防災のどれにも通じる観点として、問題顕在化、制度形成、反対論、執行、副作用、再修正を確認してください」
- 「疑惑・報道・政府説明・国会追及・司法判断を必ず分けてください」
- 「人名、組織名、役職、日付の表記揺れを正規化してください」
- 「省庁の部局レベルまで追える場合は追ってください」
- 「event_source の要約は 300〜500 字で簡潔に書いてください」
- 「リンクは疎でよいので、根拠の強いものだけにしてください」
- 「予算・税制・自治体・海外・司法・民間の観点が空白になっていないか確認してください」
