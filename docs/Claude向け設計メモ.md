# Claude 向け設計メモ

この一式は、Claude Code の設計思想に合わせて構成している。

## 1. CLAUDE.md
毎セッション読まれる恒常指示として使う。
ここには、政策・国政調査の原則、優先ソース、イベントの切り方、表現ルール、10 テーマ横断の観点を書く。

## 2. Skills
Skills は再利用ワークフローとして分割する。
今回の分割は次の通り。
- intake-source-batch
- collect-events
- link-events
- qa-event-db
- build-timeline-view

## 3. サブエージェント
探索、正規化、リンク、可視化を分離している。
大量の探索ログや試行錯誤を親コンテキストへ持ち込みにくくするため。

## 4. hooks
hooks はモデル外の決定論的ガードレール。
ここでは軽い書き込み警告と NDJSON の簡易検査だけ入れている。
本格運用では、schema validation や機密パス遮断を強化してよい。

## 5. この最終版で追加したこと
- 10 の想定テーマを横断する共通観点を `CLAUDE.md` に昇格
- 政府・国会だけでなく、自治体、司法、海外、民間、当事者団体の探索を標準動作にした
- 予算、執行、副作用、再修正まで拾うようにした
