# 日本の国政調査ガイド

このガイドは、Claude Code を使って日本の国政・行政・政策・政治問題を調べるときに、**どの種類の資料をどう優先し、どうイベント化するか**をまとめたものです。

重要なのは、**国政上の重要イベントは国会や官邸の中だけで起きるわけではない**ということです。事故、企業対応、自治体の先行措置、海外規制、訴訟、研究報告、世論の高まりなどが、後の法案・答弁・制度改正・行政処分につながることは多いです。したがって、公式制度過程と、その外側の入力や圧力を両方拾います。

また、このスターターは次の 10 テーマを想定して作られています。
- 少子化対策・子育て支援
- 物価高対策・家計支援
- 安全保障・防衛・防衛産業
- 外国人労働者・移民政策
- 生成AI・プラットフォーム規制・デジタル政策
- 医療・介護・社会保障改革
- エネルギー政策・原発・再エネ・GX
- 教育政策・無償化・大学改革・いじめ対策
- 農業政策・食料安全保障・コメ政策
- 災害対応・防災・復興政策

## 1. まず何を集めるか

テーマを受け取ったら、最初に次の観点で探索する。

### 政府・官邸
- 首相官邸
- 内閣官房
- 内閣府
- 各省庁・外局・庁
- 関連する独立行政委員会や監督当局

### 国会
- 国会会議録
- 衆議院・参議院の委員会情報
- 法案情報
- 配布資料、参考人資料、附帯決議、採決結果

### 制度会議・審議会
- 審議会
- 有識者会議
- 検討会
- WG、研究会、部会
- 中間整理、報告書、配布資料、議事要旨

### 司法・監督
- 裁判所
- 会計検査院
- 公正取引委員会
- 個人情報保護委員会
- 消費者庁
- 金融庁
- 総務省、警察庁など関係当局

### 制度の外側にある重要ソース
- 企業・プラットフォームの公式リリース、規約変更、第三者委員会報告
- 業界団体、被害者団体、NPO、学会、研究者の提言や調査
- 地方自治体の条例、独自措置、要望書、調査結果
- 海外当局、EU、米国議会、国際機関などの公表資料
- 信頼できる報道機関の調査報道、原資料付き報道

## 2. どの順番で重みづけするか

原則として、次の順で優先する。

1. 法令・告示・通知・議事録・公式会見録などの一次資料
2. 報告書・白書・検査報告・審議会資料・裁判資料・第三者委員会資料
3. 本人・政党・企業・団体・自治体・国際機関の公式発信
4. 信頼できる報道
5. 解説や評論

ただし、テーマによっては制度外の一次資料が出発点になる。たとえば、事故調報告、企業の障害報告、自治体の先行条例、海外規制当局の決定、被害者団体の調査などである。その場合も、**その資料が政策形成にどうつながったか**を後で追えるようにしておく。

## 3. テーマ横断で見るべき層

どのテーマでも、次の 5 層をなるべく揃える。
1. 問題顕在化
2. 論点形成
3. 公式制度過程
4. 執行・実施・現場反応
5. 事後評価・修正

## 4. イベントの切り方

イベントは大きすぎても小さすぎても使いにくい。以下の単位を目安にする。

### 公式制度過程のイベント
- 一つの法案提出
- 一つの閣議決定
- 一つの通知・告示・通達発出
- 一回の審議会開催または報告書公表
- 一つの国会質疑や重要答弁
- 一つの行政処分や勧告
- 一つの提訴、判決、上訴
- 一つの公式声明や会見
- 一つの予算措置や補正予算決定

### 制度の外側にあるイベント
- 重大事故や被害の発生
- 企業・プラットフォームの規約変更やサービス変更
- 業界団体の提言や自主ルール策定
- 被害者団体・NPO・学会・研究者の調査公表
- 自治体の条例制定、意見書、要望
- 海外当局の規制強化や国際機関の勧告
- 重要な調査報道の公開
- 統計公表、需給逼迫、価格高騰、出生動向など問題認識を押し上げる指標変化

### 別イベントに分けるべき例
- 同じ論点でも、被害発生と政府答弁は別
- 同じ論点でも、審議会の開催と報告書公表は別
- 報道発生と行政処分は別
- 企業の自主対応と法案提出は別
- 海外規制決定と日本政府の反応は別

## 5. リンクの考え方

イベント間リンクは、単に近い話題だから張るのではなく、**どういう経路で影響したか**まで説明できるときに張る。

主なリンクの例:
- 被害・事故 → 国会追及
- 調査報道 → 省庁調査開始
- 海外規制 → 国内審議会で参照
- 自治体の先行措置 → 国の制度設計で引用
- 企業対応 → 監督当局の評価や処分
- 裁判所判断 → 行政運用や立法議論への反映
- 統計悪化・需給逼迫・価格上昇 → 予算措置や対策本部設置

## 6. 要約の書き方

イベントと資料のペアごとの要約では、次を入れる。
- 何が起きたか
- 誰が関わったか
- 制度上どういう位置づけか
- その資料がなぜそのイベントの証拠として重要か
- 政策形成や政治判断との関係があるなら、その接点

避けるべきこと:
- 論評に寄りすぎること
- 断定が強すぎること
- 推測を事実のように混ぜること
- 「外部要因だから脇役」と決めつけること

## 7. よくある落とし穴

- 同名の委員会や会議を取り違える
- 政党名変更、会派変更、役職交代を見落とす
- 報道日と出来事の日付を混同する
- 国会での発言と政府の公式決定を同じ重さで扱う
- 省庁の部局レベルと省本体レベルを区別しない
- 外部イベントと制度イベントを一つにまとめてしまう
- 海外事例を、日本側で実際に参照されたか不明のまま強く結びつける
- 予算、執行、副作用、再修正を拾い忘れる

## 8. 幅広く調べるための原則

このスターターは日本の国政を優先するが、**それだけに限定しない**。
必要に応じて、次も取り込む。
- 地方自治体
- 海外当局
- 国際機関
- 研究者・専門家の一次資料
- 海外プラットフォームや企業の対応
- 事件・事故・災害・市場動向・技術変化

ただし、日本の制度過程に直接関係しない資料は、補助資料として位置づける。逆に、日本の制度過程に直接関係するなら、政府資料でなくても重要イベントとして扱う。
